# 数据字典

## 数据来源说明

项目默认读取：

```text
data/sample/sample_steam_games.csv
```

数据可以来自课程准备的 Steam 游戏 CSV，也可以通过页面上传用户自己的 CSV。系统会尽量兼容常见字段名，但数据质量会影响分析结果。

## 原始字段说明

| 字段 | 含义 |
|---|---|
| app_id | Steam 应用 ID |
| name | 游戏名称 |
| release_date | 发行日期 |
| price | 价格 |
| genres | 游戏类型 |
| tags | 用户标签或游戏标签 |
| positive_reviews | 好评数量 |
| negative_reviews | 差评数量 |
| developers | 开发者 |
| publishers | 发行商 |
| recommendations | 推荐数 |
| average_playtime_forever | 平均游玩时长 |
| peak_ccu | 峰值在线人数 |
| dlc_count | DLC 数量 |
| metacritic_score | Metacritic 评分 |

## 衍生字段说明

| 字段 | 生成规则 |
|---|---|
| total_reviews | positive_reviews + negative_reviews |
| positive_rate | positive_reviews / total_reviews；total_reviews 为 0 时为空值 |
| release_year | 从 release_date 提取年份 |
| is_indie | genres 或 tags 包含 Indie 时为 True |
| price_level | Free、Low、Medium、High |
| review_level | Low、Medium、High、Very High |
| genre_list | 将 genres 拆分为列表 |
| tag_list | 将 tags 拆分为列表 |

## 字段兼容规则

| 常见字段名 | 标准字段名 |
|---|---|
| Name | name |
| Release date | release_date |
| Price | price |
| Genres | genres |
| Tags | tags |
| Positive | positive_reviews |
| Negative | negative_reviews |
| Recommendations | recommendations |
| Average playtime forever | average_playtime_forever |
| Peak CCU | peak_ccu |
| DLC count | dlc_count |
| Metacritic score | metacritic_score |

字段名还会自动去除空格、转小写、将空格和连字符替换为下划线。

## 缺失值处理策略

- name 缺失：删除该行。
- price 转换失败：设为空值。
- positive_reviews 缺失：设为 0。
- negative_reviews 缺失：设为 0。
- genres、tags、developers、publishers 缺失：填充 Unknown。
- windows、mac、linux 缺失：不强制生成，相关统计跳过。
- 重复行：优先按 app_id 去重，没有 app_id 时按 name 去重。

## 关键字段处理方式

- 评论数：用于衡量玩家反馈量，但不等同于销量。
- 好评率：用于估计玩家接受度，但无评论游戏的 positive_rate 为空，不强行设为 0。
- 价格：用于划分 Free、Low、Medium、High。
- 年份：用于发行趋势和市场增长观察。

## 数据使用边界

- Steam 数据不能代表整个游戏市场。
- 评论数不等于销量。
- 好评率不等于商业成功。
- Opportunity Score 仅用于早期立项参考。
- AI 报告不能替代真实市场调研。
