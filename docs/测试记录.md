# 测试记录

## 一、测试目的

本文件记录 GameScope_V2 在课程提交前的核心测试范围、测试命令与结果，用于证明项目具备基本可运行性、结构清晰度和演示稳定性。

## 二、测试环境

- 操作系统：本地开发环境
- Python：Python 3
- 后端：FastAPI
- 数据处理：pandas
- 前端：HTML / CSS / Vanilla JavaScript
- 图表：ECharts
- 测试工具：pytest

## 三、安装与启动命令

macOS / Linux：

```bash
python3 -m pip install -r requirements.txt
python3 -m uvicorn backend.main:app --reload
```

Windows：

```bash
python -m pip install -r requirements.txt
python -m uvicorn backend.main:app --reload
```

## 四、pytest 测试命令

macOS / Linux：

```bash
python3 -m pytest -q
```

Windows：

```bash
python -m pytest -q
```

## 五、测试覆盖范围

### 1. 数据读取测试

- 默认样例数据读取
- 上传 CSV / TSV / XLSX / JSON
- 不支持扩展名时返回清晰错误
- 文件读取失败时不破坏当前已加载数据

### 2. 数据清洗测试

- 字段标准化
- 缺失值处理
- 重复数据去重
- 评论数、好评率、年份、Indie 等衍生字段生成
- 缺少 `name` 时使用其他字符串列或 `id` 列生成 `display_name/name`

### 3. 基础分析测试

- 核心指标
- Genre 分布
- 市场筛选
- 散点图数据构造

### 4. Dashboard 与系统接口测试

- `/api/dashboard/metrics`
- `/api/dashboard/insights`
- `/api/dashboard/release-trend`
- `/api/dashboard/genre-distribution`
- `/api/dashboard/tag-frequency`
- `/api/dashboard/price-distribution`
- `/api/dashboard/reception-distribution`
- `/api/system/submission-readiness`

### 5. Visual Explorer 测试

- `/api/explorer/filter`
- `/api/explorer/charts`

验证筛选条件为空或结果为空时仍返回稳定结构。

### 6. Data Q&A 测试

- 问题分类
- 无法识别问题时的提示
- 规则问答结果结构
- 多轮问答回退逻辑

### 7. Idea Lab / PLUS 测试

- 创意解析 fallback
- 相似竞品识别
- 机会评分
- 差异化建议
- Project Brief
- 顾问追问逻辑

### 8. LLM fallback 测试

- 未配置环境变量时进入规则模式
- LLM 调用失败时不导致项目崩溃

### 9. 通用 CSV 鲁棒性测试

- 上传仅包含 `id/category/value/date` 的通用 CSV
- 仍可完成读取、预览和清洗报告
- 不因缺少 Steam 专用字段导致后端崩溃

## 七、演示稳定性相关结论

- 默认样例数据可直接用于演示，无需额外下载
- 未配置 LLM 时仍可完成完整流程演示
- 自定义上传字段不标准时，系统会显示兼容性提示而不是直接报错退出
- 首页课程验收面板可直接展示当前数据规模、模块状态、文档状态和测试命令
